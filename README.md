

## Задание 1. HTML-шаблон документа

Файл **`template.html`** — шаблон заключения по результатам медицинского освидетельствования (по документу Документ.pdf).

- Вставки в формате `{}`: данные пациента (`client.*`) и перебор мероприятий обращения (`event.actions`).
- Синтаксис шаблона: `{for: action in event.actions}`, `{:end}`, `{if: ...}`, `{:elif: ...}`, `{:else}`, `{:end}`.
- Поля: ФИО, пол, дата рождения, адрес; блок «Связанные документы» — цикл по `action` (название, дата выполнения, значение).

## Задание 2. Интерфейс списка пациентов (Python + PyQt)

**Запуск:**
```bash
pip install -r requirements.txt
python patients_app.py
```

- Чтение данных из файла **`clientList`** (JSON).
- В таблице отображаются только записи с `deleted != 1`.
- Контекстное меню (ПКМ по строке):
  - **«Сформировать выходную форму»** — диалог с данными пациента (ФИО, фамилия И.О., имя, отчество, дата рождения).
  - **«Удалить строку»** — в `clientList` у выбранной записи проставляется `deleted = 1`, строка исчезает из таблицы.

## Задание 3. Работа с API

**Скрипт:** `api_client.py`

- Отправка POST на `https://test/client` с заголовками `Content-Type: application/json`, `Authorization: Token`.
- Параметры: `lastName`, `firstName`, `patrName`, `birthDate` (строка в формате `2019-11-11T11:16:32`), `status` (bool).
- Обработка ответа: проверка кода 200/201, разбор JSON, вывод в читаемом виде.
- Обработка ошибок: таймаут, ошибки соединения, невалидный JSON; при ошибке сервиса поле `data` может отсутствовать.

**Запуск:**
```bash
pip install requests
python api_client.py
# или с данными пациента в JSON:
python api_client.py "{\"fio\":\"Иванов Иван Иванович\",\"birth_date\":\"01.01.1990\"}"
```
